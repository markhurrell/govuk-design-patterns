Floating boxes

Floating boxes are areas of the interface that remain fixed to the viewport while other areas of content can pass underneath them. They allow the user to work with two pages contexts simultaneously, and usually contain interactive filtering and sorting mechanisms that effect how the content in the other page context is visible.

A good floating box design

  • makes it clear that the user can interact with content in both the box, and the content behind it
  • provides a subtle visual cue that the content underneath the box can be scrolled independently from the browser page

Floating box design proposal

The floating box should have a standard white background, so not to affect the colour-contrast of any component that might be placed inside it. Use a simple 5px wide, semi-opaque black bar overlaying the other areas of content to imply layering and depth, without veering too far into skeumorphism. There shouldn't be a shade over the other page content.

Modal dialogues

Modal dialogues are used to interrupt the user from the task they are trying to complete. Due to a long history of being abused by advertisers they are often ignored by users, and should only be used as a last resort. On GOV.UK, they are primarily used to draw attention to things that have changed on the page while the user has been away from it, that might affect their ability to complete their task, eg; session timeouts.

A good modal dialogue design

  • is visually prominent to draw the user attention
  • is strongly branded according to the site the user is visiting, so don't mistake it for an advert
  • is clearly related to the task the user is trying to complete, so they know it is relevent
  • has obvious ways to close it

Modal dialogue design proposal

The modal dialogue should be in GOV.UK blue, with a black branding area at the top. It should be centred in the screen, and shouldn't be much wider than an average mobile screen width (to stop us putting too much content in them). It should have a shade masking the rest of the page content, and a consistent shadow style to the fixed panel design.

Modal tooltip

Modal tooltips are used to provide snippets additional information within an interface. Tooltips are not a core part of the part design, and so should only be used as hints. They should be avoided in situations where the majority of users will need to know the information contained within them. For this reason, their main use is for interfaces that are used frequently and repeatedly. They are very rarely applicable for the general public.

A good modal tooltip design

  • is visually prominent to draw the user attention
  • is closely associated with the element that opened it
  • has an obvious way to close it
  • is activated and closed by affirmative click or touch actions, not by hovering

Modal tooltip design proposal

The modal tooltip should be black with white text, and ideally should only be a few words. It should be positioned below the element that opened the tooltip, and should have an arrow to make the association between elements clear. It should have a shade masking the rest of the page content, and a consistent shadow style to the fixed panel design.

Shades

Shades are a layer between a modal box and the page content beneath. They are used to increase focus on the modal by obscuring the visibility of the rest of the page content.

A good shade design

  • makes it obvious to the user that they can't interact with the background page contents
  • makes it obvious to the user that they haven't left the context of the page
  • provides a high contrast between the modal box and the background content, so that the user can see where they are meant to be looking

Shade design proposal

Full screen element behind the modal, with a semi-transparent white background to increase the contrast against the black modal element. Also utilise new CSS backdrop-filter styles to blur and reduce the contrast of the background.
